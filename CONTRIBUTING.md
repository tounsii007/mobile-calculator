# Contributing to Mobile Calculator

We love your interest in contributing to the Mobile Calculator project! Here are some guidelines to help you get started:

## How to Contribute

### Reporting Issues
- Before reporting a bug, please check the existing issues to see if your problem has already been reported.
- Provide as much detail as possible, including steps to reproduce and any relevant screenshots.

### Submitting Code Changes
1. **Fork the repository**: Create a personal copy of the repository by forking it.
2. **Create a feature branch**: Use a descriptive name for your branch (e.g., `feature/add-new-functionality`).
3. **Make your changes**: Implement the feature or bug fix in your branch.
4. **Commit your changes**: Write a clear commit message that explains your changes.
5. **Push to your branch**: Push your changes to your fork on GitHub.
6. **Create a pull request**: Go to the main repository and create a pull request, detailing your changes.

### Code Style
- Follow the existing coding style in the project. Consistency is key!
- Use appropriate naming conventions for functions and variables.

### Testing
- Ensure all new features are accompanied by tests.
- Run existing tests to ensure nothing is broken by your changes.

## Need Help?
If you have questions or need guidance, feel free to open an issue or ask in our community channels.

Thank you for contributing to Mobile Calculator!
